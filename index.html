Why this is fundamentally different from typical automation

The bottleneck is not execution speed — it is decision quality. Deciding which sub-model to retrain, what data to collect, how to adjust thresholds, and why an approach failed requires institutional knowledge accumulated over months. Scripts can automate execution, but they cannot automate the expert judgment that makes each pipeline run successful. This is the gap AAICLAW fills.

Compound Value

It Gets Smarter Every Time It Runs

Unlike traditional automation, AAICLAW accumulates institutional knowledge. Each pipeline run — and each human correction — makes the next run faster and more accurate.

Run 1–5: Cold Start ~30% time saved

The virtual team follows general best practices. Humans correct frequently. But every correction is captured as a permanent experience entry.

Run 5–15: Accumulation ~60% time saved

The system starts making connections: "This alert resembles run-202604. That run succeeded with approach X." Human corrections decrease.

Run 15+: Institutional Expert ~80% time saved

Comprehensive knowledge base of attack types, regional quirks, annotation pitfalls, and threshold patterns. A new team member is productive on day one.

"Every human correction becomes a permanent lesson. The system never forgets."

How Experience Memory works

After each pipeline run, a retrospective is written to structured Experience Memory — what worked, what failed, why, and what to try next time. When a new alert arrives, the AI Researcher queries this memory before proposing a strategy. Over time, the memory becomes a queryable knowledge base covering:

  • Which approaches work for which attack types
  • Which regions have quirks (template changes, unusual document quality)
  • Common annotation pitfalls and how to detect them
  • Threshold sensitivity patterns per card type
  • Historical annotation team accuracy patterns

How It Operates

AI-Human Collaboration, Not Replacement

Three tiers of AI models paired with matching human expertise levels. Senior engineers develop the playbook; cost-efficient models execute daily tasks; the system self-improves through structured feedback loops.

Three-Tier Organization

Each tier uses the right AI model at the right cost, paired with the right human role.

Tier 0 — Manager

Strategic Oversight

Claude Tech Lead

Full permissions. Manages team SOPs, agent configurations, and organizational knowledge. Summarizes Mentor discoveries into permanent institutional memory. The single source of truth.

Tier 1 — Mentor

Skill Development

Claude Code Senior Engineer

Develops and refines agent skills through interactive sessions. Reviews agent execution logs daily to identify improvements. Read access to production; writes go through code review.

Tier 2 — Agent

Daily Execution

Gemini Operator / User

Executes scheduled and on-demand tasks in a sandboxed environment. Limited to assigned skills and read-only cluster access. Escalates to humans when encountering permission boundaries.

Self-Improving Feedback Loop

1. Agents execute daily tasks, recording full session logs

2. Mentors analyze sessions: where did the agent waste tokens? Which skill instructions were ambiguous?

3. Mentors refine skills and prompts based on findings

4. Manager reviews and merges improvements into the system

5. Next day, agents run with improved instructions

Cost Structure

Daily agent execution uses cost-efficient models (Gemini) for routine tasks like data collection, monitoring, and training orchestration.

High-capability models (Claude) are reserved for skill development and session analysis — activities that require deep reasoning but happen less frequently.

~$250–500/month total AI cost

vs. $15–20K/month for equivalent human team
